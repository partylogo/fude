Mobile App Design Guideline

一、設計總則 (Design Principles)

設計哲學

• 簡潔為主：追求清晰直觀的介面設計，避免不必要的視覺干擾

• 以用戶為中心：優先考慮用戶需求和使用情境，提供直觀的操作體驗

• 可預測的互動：遵循平台慣例，確保操作邏輯的一致性

• 無障礙設計：考慮不同能力用戶的需求，提供可及性支援

品牌調性

融合懷舊古樸與現代實用的特質，呈現溫暖、專業、可信賴的品牌形象。採用低飽和度的暖色調，營造舒適的日常氛圍。

二、視覺設計 (Visual Design)

1. 色彩系統 (Color System)

採用極簡的雙色核心系統，基於深赭紅和煙燻灰的延伸變化，營造溫暖古樸且現代的設計氛圍。所有色彩變化都基於明度調整和透明度變化，確保完美的視覺和諧。

#F7F3E8

主要背景色

色彩功能明確分工

• 深赭紅系列：積極行動（主按鈕、購買、確認、品牌標識）

• 煙燻灰系列：所有系統狀態（錯誤、警告、次要按鈕、成功提示）

• 米杏白系列：內容承載（背景、表面、留白區域）

核心原則：紅色 = 行動召喚，灰色 = 系統回饋，分工明確不混亂

純粹雙色哲學

• 2 核心色彩：深赭紅 + 煙燻灰，不再有任何中間變化色

• 互動反饋：使用陰影、位移、透明度創造懸停效果

• 透明度變化：使用 rgba 創造微妙的狀態區分

• 極致簡潔：比 Airbnb 還要純粹的色彩系統

• 認知清晰：紅色專門引導行動，灰色專門傳達資訊

2. 字體與排版 (Typography)

採用系統字體確保最佳效能與可讀性，iOS 使用 San Francisco，Android 使用 Roboto。

3. 圖標與圖像 (Icons & Imagery)

圖標採用 24px 標準尺寸，線條粗細 2px，符合 Material Design 規範。圖像風格偏向溫暖寫實，避免過度飽和的色彩。

四、互動行為 (Interaction Design)

觸控反饋

• 點擊：提供視覺反饋，按鈕按下時改變顏色和陰影

• 長按：顯示上下文選單或額外選項

• 滑動：支援左右滑動切換頁面或項目

• 拖曳：用於排序或移動項目

狀態設計

• 載入狀態：顯示載入動畫，告知用戶系統正在處理

• 空狀態：提供友善的空白狀態提示

• 錯誤狀態：清楚說明錯誤原因和解決方案

動畫原則

• 使用緩入緩出 (ease-in-out) 動畫曲線

• 動畫持續時間控制在 200-500ms 之間

• 避免過度動畫影響使用者體驗

• 提供關閉動畫的選項以增進無障礙性

響應式設計

支援多種螢幕尺寸，確保在不同裝置上都能提供良好的使用體驗。

斷點設計

• 小螢幕 (< 768px)：單欄布局，按鈕全寬

• 中螢幕 (768px - 1024px)：雙欄布局，適度間距

• 大螢幕 (> 1024px)：多欄布局，最大寬度限制

六、平台一致性與差異 (Platform Specifics)

iOS vs Android 差異處理

• 導覽模式：iOS 使用 Tab Bar，Android 使用 Bottom Navigation

• 返回操作：iOS 使用左上角返回按鈕，Android 使用系統返回鍵

• 字體選擇：iOS 使用 San Francisco，Android 使用 Roboto

• 圓角設計：iOS 偏好較大圓角，Android 使用適中圓角

/* 平台特定樣式 */ .ios-style { border-radius: 16px; font-family: -apple-system, BlinkMacSystemFont; } .android-style { border-radius: 8px; font-family: Roboto, sans-serif; }

九、無障礙設計 (Accessibility)

色彩對比

簡化色彩系統後的對比度檢測結果：

• 深赭紅 (#8A4A45) 在白色背景上：對比度 5.2:1 ✓

• 古銅黃 (#B8884F) 在白色背景上：對比度 4.8:1 ✓

• 煙燻灰 (#2F2B27) 在白色背景上：對比度 12.1:1 ✓

• 白色文字在深赭紅背景上：對比度 5.2:1 ✓

所有組合均符合 WCAG 2.1 AA 標準

色彩依賴性降低

• 成功/警告/錯誤狀態除了色彩外，還配合圖標和文字說明

• 重要操作按鈕除了顏色區分外，還有位置和大小的層級區分

• 狀態變化提供多重感官反饋（視覺+觸覺+聲音）

觸控友善設計

• 最小觸控面積：44px × 44px (iOS) / 48dp × 48dp (Android)

• 按鈕間最小間距：8px

• 支援語音控制和輔助觸控

• 動態字型大小調整支援

十、效能最佳化 (Performance)

色彩系統效能優勢

• 減少色彩計算：從5色簡化為4色系統，減少CSS變數處理

• 統一暖色調：減少GPU渲染時的色彩空間轉換

• 語義化命名：提高開發效率，減少錯誤使用

動畫效能

• 優先使用 CSS transform 和 opacity

• 避免觸發重排 (reflow) 和重繪 (repaint)

• 使用 will-change 屬性優化動畫層

• 統一的色彩變化減少重繪成本

/* 效能最佳化的色彩動畫 */ .color-transition { background-color: var(--primary-color); transition: background-color 0.3s ease; } .color-transition:hover { background-color: #75403B; /* 預計算的hover色彩 */ }
